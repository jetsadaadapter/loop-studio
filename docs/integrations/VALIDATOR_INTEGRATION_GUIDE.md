# Validator Integration Guide

## Overview

This guide shows **where and when** to integrate validators in the Meta-Prompt analysis pipeline.

## Integration Points

### 🔴 Point 1: Meta-Prompt Config Generation (PROMPT A)

**Location:** API Route หรือ Server Action ที่รับ user input และส่งไปให้ LLM generate config

**File:** `src/app/api/tools/meta-prompt/route.ts` (ตัวอย่าง - อาจต้องสร้างใหม่)

```typescript
import { validateMetaPromptConfigFull } from "@/core/validators/meta-prompt-config.validator";

export async function POST(request: Request) {
  const { userInput } = await request.json();

  // Step 1: Call Meta-Prompt LLM
  const rawLLMOutput = await callMetaPromptLLM(userInput);

  // Step 2: Parse JSON
  let configJson;
  try {
    configJson = JSON.parse(rawLLMOutput);
  } catch (error) {
    return Response.json(
      { error: "LLM returned invalid JSON" },
      { status: 500 }
    );
  }

  // 🔴 VALIDATION POINT 1
  const validation = validateMetaPromptConfigFull(configJson);

  if (!validation.success) {
    console.error("Meta-Prompt Config Validation Failed:", validation.errors);
    
    return Response.json(
      {
        error: "Invalid configuration generated by LLM",
        details: validation.errors,
        rawConfig: configJson, // For debugging
      },
      { status: 400 }
    );
  }

  // Step 3: Use validated config to create Apify job
  const validConfig = validation.data;
  
  const apifyJob = await createApifyAnalysisJob({
    startUrls: validConfig.input.startUrls,
    prompt: validConfig.config.prompt,
    tools: validConfig.config.tools,
    blueprint: validConfig.blueprint,
  });

  return Response.json({
    success: true,
    jobId: apifyJob.id,
    config: validConfig.preview, // Return preview to user
  });
}
```

### 🔴 Point 2: Analyzer Output Processing (PROMPT B)

**Location:** Component ที่แสดงผลลัพธ์จาก Analyzer LLM

**File:** `src/app/tool/[id]/components/visualizer/tabs/tab-output-overview.tsx`

```typescript
import { validateAnalyzerOutputFull } from "@/core/validators/analyzer-output.validator";
import { generateDynamicLayoutFromSchema } from "./tab-output-dynamic-schema";

export function TabOutputOverview({ job }: { job: ToolJob }) {
  const [validationError, setValidationError] = useState<string[] | null>(null);

  // Check if result is structured report
  const isStructuredReport = useMemo(() => {
    return Boolean(
      job.result &&
      typeof job.result === "object" &&
      !Array.isArray(job.result) &&
      "section_meta" in job.result &&
      "section_rows" in job.result
    );
  }, [job.result]);

  // 🔴 VALIDATION POINT 2
  const validatedOutput = useMemo(() => {
    if (!isStructuredReport) return null;

    const validation = validateAnalyzerOutputFull(job.result);

    if (!validation.success) {
      console.error("Analyzer Output Validation Failed:", validation.errors);
      setValidationError(validation.errors);
      return null;
    }

    setValidationError(null);
    return validation.data;
  }, [job.result, isStructuredReport]);

  // Show validation errors
  if (validationError) {
    return (
      <div className="p-4 border border-red-200 bg-red-50 rounded-lg">
        <h3 className="text-sm font-bold text-red-900 mb-2">
          ⚠️ Validation Failed
        </h3>
        <ul className="text-xs text-red-700 space-y-1">
          {validationError.map((err, idx) => (
            <li key={idx}>• {err}</li>
          ))}
        </ul>
        <details className="mt-3">
          <summary className="text-xs font-semibold cursor-pointer">
            Show Raw Output
          </summary>
          <pre className="mt-2 text-xs bg-white p-2 rounded overflow-auto max-h-60">
            {JSON.stringify(job.result, null, 2)}
          </pre>
        </details>
      </div>
    );
  }

  // Generate dynamic layout from validated output
  if (validatedOutput) {
    const dynamicItems = generateDynamicLayoutFromSchema(job, [validatedOutput]);
    return <DynamicLayoutVisualizer items={dynamicItems} />;
  }

  // Fallback to existing visualizers
  return <ExistingVisualizerLogic job={job} />;
}
```

## Integration Timeline

### Phase 1: Backend (Meta-Prompt Config) ✅ ควรทำก่อน

1. สร้าง API Route: `/api/tools/meta-prompt`
2. เพิ่ม validation ก่อนส่งเข้า Apify
3. Return errors พร้อม details ให้ user

**Priority:** 🔥 High - ป้องกัน invalid config เข้า pipeline

### Phase 2: Frontend (Analyzer Output) ✅ ควรทำต่อ

1. เพิ่ม validation ใน `tab-output-overview.tsx`
2. แสดง error UI เมื่อ validation fail
3. Log errors สำหรับ debugging

**Priority:** 🔥 High - ป้องกัน UI crash จาก malformed data

### Phase 3: Monitoring & Logging ⏳ Optional

1. Track validation failure rate
2. Log common error patterns
3. Auto-report to error tracking service

**Priority:** 🟡 Medium - ช่วย improve LLM prompts

## Example Integration Scenarios

### Scenario 1: Meta-Prompt LLM Returns Invalid Config

**What happens:**
```typescript
// LLM generates config with wrong section_id format
{
  "blueprint": {
    "sections": [
      {
        "section_id": "section1",  // ❌ Should be "s1"
        ...
      }
    ]
  }
}
```

**Validator catches:**
```
[blueprint.sections.0.section_id] section_id must be s1, s2, s3, etc.
```

**User sees:**
```json
{
  "error": "Invalid configuration generated by LLM",
  "details": [
    "[blueprint.sections.0.section_id] section_id must be s1, s2, s3, etc."
  ]
}
```

**Next steps:**
- Retry with refined prompt
- Show error to user with option to retry
- Log for prompt improvement

---

### Scenario 2: Analyzer LLM Returns Nested Arrays

**What happens:**
```typescript
// LLM generates output with nested arrays
{
  "section_rows": [
    {
      "tags": ["tag1", ["nested", "array"]]  // ❌ FORBIDDEN
    }
  ]
}
```

**Validator catches:**
```
Nested array detected at section_rows[0].tags — FORBIDDEN (use comma-separated strings instead)
```

**UI shows:**
- ⚠️ Validation error banner
- Collapsible raw JSON output
- Fallback to raw JSON viewer

            section_type: "bar_chart",
            what_to_measure: "Frequency",
            signal_keywords: "keywords",
            priority: 2,
            labels: {
              intent_group: "กลุ่ม"
            }
          }
        ],
        overall_sentiment_focus: "mixed",
        confidence_note: "High confidence"
      }
    };

    const result = validateMetaPromptConfigFull(validConfig);
    expect(result.success).toBe(true);
  });

  it("should reject config with unresolved placeholders", () => {
    const invalidConfig = {
      // ... valid fields
      config: {
        prompt: "Goal: [fill: preview.goal]"  // ❌ Unresolved
      }
    };

    const result = validateMetaPromptConfigFull(invalidConfig);
    expect(result.success).toBe(false);
    expect(result.errors).toContain(
      expect.stringContaining("unresolved [fill:...] placeholders")
    );
  });

  it("should reject config with label-property mismatch", () => {
    const invalidConfig = {
      // ... valid fields
      blueprint: {
        sections: [
          {
            labels: { intent_group: "Label" }  // Has intent_group
          }
        ]
      },
      config: {
        tools: {
          function_declarations: [
            {
              parameters: {
                properties: {
                  different_key: {}  // ❌ No intent_group
                }
              }
            }
          ]
        }
      }
    };

    const result = validateMetaPromptConfigFull(invalidConfig);
    expect(result.success).toBe(false);
  });
});

## Error Handling Best Practices

### 1. Don't Fail Silent

```typescript
// ❌ Bad: Silent failure
const result = validateMetaPromptConfigFull(config);
if (!result.success) {
  // Use config anyway
  await createJob(config);
}

// ✅ Good: Explicit error handling
const result = validateMetaPromptConfigFull(config);
if (!result.success) {
  console.error("Validation failed:", result.errors);
  throw new Error("Invalid config");
}
```

### 2. Provide Context in Errors

```typescript
// ❌ Bad: Generic error
return Response.json({ error: "Validation failed" }, { status: 400 });

// ✅ Good: Detailed error
return Response.json(
  {
    error: "Invalid configuration generated by LLM",
    details: validation.errors,
    help: "Try rephrasing your request or check the example format"
  },
  { status: 400 }
);
```

### 3. Log for Debugging

```typescript
// Always log validation failures
if (!validation.success) {
  console.error("Validation failed:", {
    errors: validation.errors,
    input: userInput,
    rawConfig: configJson,
    timestamp: new Date().toISOString()
  });
}
```

## Performance Considerations

### Validation Cost

- **Meta-Prompt Config:** ~2-5ms
- **Analyzer Output:** ~3-8ms
- **Total overhead:** <10ms (negligible)

### When to Skip Validation

**Never skip in production.** But for development/testing:

```typescript
// Environment flag
const SKIP_VALIDATION = process.env.SKIP_VALIDATION === "true";

if (!SKIP_VALIDATION) {
  const validation = validateMetaPromptConfigFull(config);
  if (!validation.success) {
    throw new Error("Validation failed");
  }
}
```

## Troubleshooting

### Issue: "Validation always fails"

**Check:**
1. LLM prompt ให้ output ตรงตาม schema หรือไม่
2. มี unresolved `[fill:...]` placeholders หรือไม่
3. JSON parsing ถูกต้องหรือไม่

### Issue: "Labels don't match properties"

**Fix:** ตรวจสอบว่า function declaration properties keys ตรงกับ blueprint labels keys

```typescript
// Must match exactly
blueprint.sections[0].labels = { intent_group: "..." }
config.tools.function_declarations[0].parameters.properties = { intent_group: {...} }
```

### Issue: "Nested arrays detected"

**Fix:** Convert arrays to comma-separated strings

```typescript
// ❌ Wrong
tags: ["tag1", "tag2"]

// ✅ Correct
tags: "tag1,tag2"
```
